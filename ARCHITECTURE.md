# music-generator Architecture

> Authoritative architecture reference for the `music-generator` service.
> Every implementation decision in this repo must match the specifications below.
> If a code change requires deviating from this document, update this document first.

---

## 1. Overview

### Purpose

`music-generator` is an AI-assisted melody composition web app. A user types a natural-language prompt (for example, *"Create a 4 voices melody in Bach baroque style"*), the backend calls OpenAI GPT-4o to produce a melody in **ABC notation**, the result is persisted to Firestore, and the browser renders the sheet music with the **abcjs** JavaScript library. The user can edit the ABC text; the score re-renders live. Saved melodies appear in a sidebar where each entry can be loaded, saved, duplicated, or deleted.

### Business Context

This is a demonstration-class application deployed on Google Cloud Run under `demo.devops-for-hire.com`. It showcases an end-to-end GenAI pipeline (prompt → LLM → structured output → render → persist) with production-grade CI/CD, multi-environment deployment, and Firestore persistence.

### High-Level Design Goals

- Zero-install user experience (runs entirely in the browser, no plugins)
- Self-contained binary — HTML/CSS/JS embedded, abcjs loaded from a public CDN
- Clean separation of concerns: HTTP handlers → Store interface → Firestore
- Deterministic deploys via Terraform + GitHub Actions
- Graceful degradation: server never crashes on OpenAI or Firestore errors — a clear JSON error is always returned

### Technology Stack

| Layer | Choice | Notes |
|-------|--------|-------|
| Language | Go **1.22+** | Required for the new `ServeMux` routing patterns |
| HTTP | Go standard library `net/http` | `http.ServeMux` with method+path patterns |
| Persistence | Google Cloud Firestore (Native mode) | Named database `music-generator` (not `(default)`) |
| LLM | OpenAI `gpt-4o` via `github.com/sashabaranov/go-openai` | Temperature 0.8, max tokens 2000 |
| UI rendering | [abcjs](https://www.abcjs.net/) v6 (CDN) | `abcjs-basic-min.js` loaded from `cdn.jsdelivr.net` |
| Container | Alpine multi-stage Docker image | Non-root user |
| Compute | Google Cloud Run (fully managed, v2 service) | Per-environment service |
| IaC | Terraform **>= 1.6** | Required for variables inside `import` blocks |
| CI/CD | GitHub Actions | Branch-based: `stage` → staging, `main` → production |
| DNS | Google Cloud DNS, zone in `dfh-ops-id` | **CNAME** records to `ghs.googlehosted.com` |

## 3. API Endpoint Specification

All API endpoints return JSON with `Content-Type: application/json`. Error responses use the common shape:

```json
{ "error": "human-readable message" }
```

### 3.1 `POST /api/generate`

**Description** — Generate a new melody from a free-form prompt. Calls OpenAI, persists the result in Firestore, returns the created document.

**Request body:**

```json
{
  "prompt": "Create a 4 voices melody in Bach baroque style"
}
```

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `prompt` | string | yes | Non-empty, trimmed; max 2000 chars |

**Response 201 Created:**

```json
{
  "id": "qF7xS8c2bRj3mZt1vKd9",
  "title": "Create a 4 voices melody in Bach baroque style",
  "prompt": "Create a 4 voices melody in Bach baroque style",
  "abc_notation": "X:1\nT:Baroque 4-Voice Invention\nM:4/4\nL:1/8\nK:Cmaj\nV:1 clef=treble\n|: G2 A2 B2 c2 | d2 e2 f2 g2 :|\nV:2 clef=treble\n|: E2 F2 G2 A2 | B2 c2 d2 e2 :|\nV:3 clef=bass\n|: C2 D2 E2 F2 | G2 A2 B2 c2 :|\nV:4 clef=bass\n|: C,2 D,2 E,2 F,2 | G,2 A,2 B,2 C2 :|",
  "created_at": "2026-04-14T17:23:04.512Z",
  "updated_at": "2026-04-14T17:23:04.512Z"
}
```

**Status codes:**

| Code | When |
|------|------|
| 201 Created | Success |
| 400 Bad Request | Missing/empty `prompt`, malformed JSON, or prompt > 2000 chars |
| 405 Method Not Allowed | Any method other than `POST` |
| 502 Bad Gateway | OpenAI call failed or returned empty content |
| 500 Internal Server Error | Firestore write failed |

**Edge cases:**

- Prompt containing only whitespace → 400
- OpenAI returns content wrapped in ``` fences or prose → handler strips fences and surrounding prose; see §9
- OpenAI returns empty string after cleaning → 502
- Title derivation: first non-empty line of the prompt trimmed to **50 chars** (hard cap, no ellipsis); if resulting string is empty use `"Untitled melody"`

### 3.4 `PUT /api/melodies/{id}`

**Description** — Update `title` and/or `abc_notation` of an existing melody. Both fields are optional in the request; missing fields are left unchanged. `updated_at` is set server-side to `time.Now().UTC()`.

**Request body:**

```json
{
  "title": "Baroque 4-Voice Invention (revised)",
  "abc_notation": "X:1\nT:Baroque 4-Voice Invention (revised)\nM:4/4\nL:1/8\nK:Cmaj\nV:1 clef=treble\n|: G2 A2 B2 c2 :|\n"
}
```

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `title` | string | no | Max 200 chars if present |
| `abc_notation` | string | no | Max 20000 chars if present |

**Response 200 OK:** full updated document, same shape as §3.3.

```json
{
  "id": "qF7xS8c2bRj3mZt1vKd9",
  "title": "Baroque 4-Voice Invention (revised)",
  "prompt": "Create a 4 voices melody in Bach baroque style",
  "abc_notation": "X:1\nT:Baroque 4-Voice Invention (revised)\nM:4/4\nL:1/8\nK:Cmaj\nV:1 clef=treble\n|: G2 A2 B2 c2 :|\n",
  "created_at": "2026-04-14T17:23:04.512Z",
  "updated_at": "2026-04-14T17:45:19.700Z"
}
```

**Status codes:**

| Code | When |
|------|------|
| 200 OK | Success |
| 400 Bad Request | Malformed JSON, both fields missing/empty, or length limits exceeded |
| 404 Not Found | Document ID does not exist |
| 405 Method Not Allowed | Any method other than `PUT` |
| 500 Internal Server Error | Firestore write failed |

**Edge cases:**

- Empty object `{}` → 400 (must update at least one field)
- `title` is an empty string (after trim) → treated as "not provided"; if `abc_notation` also absent → 400
- `prompt` field in body → silently ignored (prompt is immutable after creation)

### 3.5 `POST /api/melodies/{id}/duplicate`

**Description** — Create a copy of an existing melody. The new document has a fresh auto-generated ID, fresh `created_at`/`updated_at`, the same `prompt` and `abc_notation`, and `title = <original title> + " (copy)"`.

**Request body:** none (empty body accepted).

**Response 201 Created:**

```json
{
  "id": "zY9xW8v7uT6sR5qP4oN3",
  "title": "Create a 4 voices melody in Bach baroque style (copy)",
  "prompt": "Create a 4 voices melody in Bach baroque style",
  "abc_notation": "X:1\nT:Baroque 4-Voice Invention\n...",
  "created_at": "2026-04-14T17:50:02.810Z",
  "updated_at": "2026-04-14T17:50:02.810Z"
}
```

**Status codes:**

| Code | When |
|------|------|
| 201 Created | Success |
| 404 Not Found | Source document does not exist |
| 405 Method Not Allowed | Any method other than `POST` |
| 500 Internal Server Error | Firestore read or write failed |

**Edge cases:**

- Duplicating an already-duplicated document produces a title like `"Foo (copy) (copy)"` — this is intentional (users expect no de-duplication of the suffix). See §9.
- Truncation: if appending `" (copy)"` pushes the title past 200 chars, the **base** title is truncated so that the resulting string is exactly 200 chars.

## 4. Data Models

### Firestore Database

- **Database name:** `music-generator` (a **named** Firestore database, not `(default)`)
- **Location:** `nam5` (multi-region)
- **Mode:** Native

### Collection: `melodies`

Document ID: Firestore **auto-generated ID** (20-character base62). The application never supplies an ID on create.

| Field | Firestore type | Go type | JSON key | Notes |
|-------|----------------|---------|----------|-------|
| (doc ID) | — | `string` | `id` | Not stored as a field; populated from `doc.Ref.ID` on read |
| `title` | String | `string` | `title` | ≤ 200 chars |
| `prompt` | String | `string` | `prompt` | ≤ 2000 chars; immutable after create |
| `abc_notation` | String | `string` | `abc_notation` | ≤ 20000 chars |
| `created_at` | Timestamp | `time.Time` | `created_at` | Server-assigned on create |
| `updated_at` | Timestamp | `time.Time` | `updated_at` | Server-updated on each write |

JSON timestamps use RFC 3339 with microsecond precision (`2026-04-14T17:23:04.512Z`).

### Indexes

The listing query `melodies order by created_at desc` is a single-field descending order by a built-in field. Firestore creates a descending single-field index automatically — **no composite index required**. No `indexes` Terraform resource is needed in v1.

### Seed Data

No seed documents are created by Terraform. The collection is empty on first deploy; the user populates it by invoking `POST /api/generate`.

## 5. Store Interface

Defined in `service/internal/store/store.go`.

```go
package store

import (
    "context"
    "time"
)

// Melody is the in-memory representation of a document in the `melodies` collection.
type Melody struct {
    ID          string    `json:"id"           firestore:"-"`
    Title       string    `json:"title"        firestore:"title"`
    Prompt      string    `json:"prompt"       firestore:"prompt"`
    ABCNotation string    `json:"abc_notation" firestore:"abc_notation"`
    CreatedAt   time.Time `json:"created_at"   firestore:"created_at"`
    UpdatedAt   time.Time `json:"updated_at"   firestore:"updated_at"`
}

// MelodyUpdate carries the partial-update payload for UpdateMelody.
// A nil pointer means "leave this field unchanged".
type MelodyUpdate struct {
    Title       *string
    ABCNotation *string
}

// ErrNotFound is returned by Get/Update/Delete/Duplicate when the document ID does not exist.
var ErrNotFound = errors.New("melody not found")

// Store is the persistence-layer contract. All methods honour the provided context.
type Store interface {
    // CreateMelody persists a new Melody. The implementation assigns ID, CreatedAt,
    // and UpdatedAt. The returned Melody reflects the stored document.
    CreateMelody(ctx context.Context, m Melody) (Melody, error)

    // ListMelodies returns all melodies ordered by CreatedAt descending.
    // An empty collection yields a non-nil zero-length slice.
    ListMelodies(ctx context.Context) ([]Melody, error)

    // GetMelody returns a single melody by ID. Returns ErrNotFound if missing.
    GetMelody(ctx context.Context, id string) (Melody, error)

    // UpdateMelody applies the partial update. The implementation sets UpdatedAt
    // to time.Now().UTC(). Returns ErrNotFound if the ID does not exist.
    UpdateMelody(ctx context.Context, id string, upd MelodyUpdate) (Melody, error)

    // DuplicateMelody reads the source document, creates a new document with the
    // same prompt and abc_notation, a title suffixed with " (copy)" (truncated to
    // 200 chars if needed), and fresh timestamps. Returns ErrNotFound if the
    // source ID does not exist.
    DuplicateMelody(ctx context.Context, id string) (Melody, error)

    // DeleteMelody removes a melody by ID. Returns ErrNotFound if missing.
    DeleteMelody(ctx context.Context, id string) error

    // DeleteAllMelodies removes every document in the collection. This is used
    // by tests and admin tooling only; it MUST use firestore.BulkWriter (never
    // the deprecated Batch()). Returns the number of documents deleted.
    DeleteAllMelodies(ctx context.Context) (int, error)

    // Close releases any underlying client resources.
    Close() error
}
```

### Firestore Implementation Notes

- Constructor: `NewFirestoreStore(ctx context.Context, projectID, databaseName string) (*FirestoreStore, error)` — uses `firestore.NewClientWithDatabase(ctx, projectID, databaseName)`.
- `CreateMelody` uses `Collection("melodies").NewDoc()` to get a client-side auto-ID, then `doc.Create(ctx, m)`.
- `ListMelodies` runs `Collection("melodies").OrderBy("created_at", firestore.Desc).Documents(ctx)` and iterates.
- `UpdateMelody` uses `firestore.Update{}` entries for each non-nil field + always `{Path: "updated_at", Value: time.Now().UTC()}`.
- **`DeleteAllMelodies` and any future bulk write path MUST use `firestore.BulkWriter`** (`client.BulkWriter(ctx)` → `bw.Delete(ref)` per document → `bw.End()`). **The deprecated `client.Batch()` is forbidden** — it is capped at 500 ops, lacks rate-limiting, and has been superseded.
- `GetMelody`, `UpdateMelody`, `DeleteMelody`, and `DuplicateMelody` translate `status.Code(err) == codes.NotFound` into `ErrNotFound`.

## 6. Server and Routing Design

### Server Struct

```go
type Server struct {
    cfg       *config.Config
    store     store.Store
    gen       openai.Generator
    indexTmpl *template.Template
}
```

Constructed once via `New(cfg, store, gen)`. The HTML template is parsed from `templates.FS` at construction time; a parse error is fatal.

### Go 1.22 ServeMux Routing

All routes use the Go 1.22 method+pattern syntax. Registered in `SetupRoutes()`:

| Pattern | Handler |
|---------|---------|
| `GET /health` | `handleHealth` |
| `POST /api/generate` | `handleGenerate` |
| `GET /api/melodies` | `handleListMelodies` |
| `GET /api/melodies/{id}` | `handleGetMelody` |
| `PUT /api/melodies/{id}` | `handleUpdateMelody` |
| `POST /api/melodies/{id}/duplicate` | `handleDuplicateMelody` |
| `DELETE /api/melodies/{id}` | `handleDeleteMelody` |
| `GET /{$}` | `handleIndex` |

Path parameters are read via `r.PathValue("id")`.

### Why `GET /{$}` and not `GET /`

**This is a hard requirement.** In Go 1.22's `ServeMux`, a pattern ending in `/` matches **any path with that prefix**. If the index handler were registered as `GET /`, then:

- a request like `DELETE /api/melodies/xyz` sent with a wrong method (e.g., `POST /api/melodies/xyz` without a matching route) could fall back to `GET /` and receive the HTML page instead of the expected `405 Method Not Allowed`.
- clients that attempt API calls on the wrong method silently get HTML, which is undebuggable.

The pattern `GET /{$}` matches **only the exact path `/`**. Any other path that fails to match an API pattern correctly produces `404 Not Found`, and any method mismatch on a registered API path correctly produces `405 Method Not Allowed` (ServeMux handles this automatically when two patterns share a path but differ on method).

### Middleware Chain

A single lightweight chain wraps the mux:

1. `recoverMW` — catches panics, logs, returns 500 JSON
2. `logMW` — structured access log (method, path, status, duration)

Applied as: `handler := recoverMW(logMW(mux))`.

### Static / Template Serving

- No static file directory. The entire UI lives in `templates/index.html` (HTML + CSS + JS inline).
- abcjs is loaded from CDN `https://cdn.jsdelivr.net/npm/abcjs@6.4.4/dist/abcjs-basic-min.js` (see §10).
- The template receives `{Version string; Environment string}` for display in the footer and version badge.

## 7. Configuration

All configuration comes from environment variables. Loading is done in `internal/config/config.go`.

| Env Var | Type | Default | Description |
|---------|------|---------|-------------|
| `PORT` | string | `"8080"` | HTTP listen port |
| `APP_VERSION` | string | `"dev"` | Semantic version injected by CI/CD |
| `ENVIRONMENT` | string | `"local"` | `"staging"`, `"production"`, or `"local"` |
| `GCP_PROJECT_ID` | string | `""` | GCP project ID. If empty, Firestore client is not created and the server returns 503 for any endpoint that requires the store. |
| `FIRESTORE_DATABASE` | string | `"music-generator"` | Firestore named database |
| `OPENAI_API_KEY` | string | `""` | OpenAI API key. If empty, `POST /api/generate` returns 503; other endpoints are unaffected. |
| `OPENAI_MODEL` | string | `"gpt-4o"` | Model override for debugging |
| `OPENAI_MAX_TOKENS` | int | `2000` | Max output tokens |
| `OPENAI_TEMPERATURE` | float64 | `0.8` | Sampling temperature |

Loading precedence: environment variable > default. There is no config file and no `flag` parsing.

`OPENAI_API_KEY` **must never be committed to git**. It is injected at runtime:

- Locally: via a `.env` file (gitignored) or shell `export`
- In Cloud Run: via the `env` block in `cloud-run.tf`, sourced from a **Secret Manager** secret `openai-api-key` (managed out-of-band; see README §Manual Setup)

## 9. Algorithm and Logic Edge Cases

### 9.1 OpenAI Response Parsing

The `openai.Generator.Generate(ctx, prompt)` call returns `(abc string, err error)`. The raw assistant message may contain:

- Pure ABC notation (happy path)
- ABC wrapped in ```abc ... ``` fences
- ABC wrapped in ``` ... ``` fences (no language tag)
- Prose before or after the notation (rare but observed)

The client applies **cleanup in this order**:

1. Trim leading/trailing whitespace.
2. If the string starts with ` ``` ` (optionally followed by a language word up to 20 non-whitespace chars), remove the opening fence line.
3. If the string ends with ` ``` `, remove the closing fence line.
4. Trim whitespace again.
5. If the result does not contain a line starting with `X:` (the ABC tune-header marker), return `ErrMalformedABC` — the handler maps this to **502 Bad Gateway**.

**Edge cases explicitly covered:**

| Input from OpenAI | Output |
|-------------------|--------|
| `""` (empty) | `ErrEmptyResponse` → 502 |
| `"   \n\n  "` (whitespace only) | `ErrEmptyResponse` → 502 |
| `"X:1\nT:Foo\nK:C\nC D E F|"` | Returned as-is |
| ```` "```abc\nX:1\nK:C\nC|\n```" ```` | `"X:1\nK:C\nC|"` |
| ```` "```\nX:1\nK:C\nC|\n```" ```` | `"X:1\nK:C\nC|"` |
| `"Here is your melody:\n\nX:1\nK:C\nC|"` | `"Here is your melody:\n\nX:1\nK:C\nC|"` — prose kept (ABC detector still finds `X:`) |
| `"Sorry, I cannot do that."` | `ErrMalformedABC` → 502 (no `X:` line) |

### 9.2 Regex Verification for the Fence Stripper

The fence-stripping code uses two regexes:

```go
var openFenceRE  = regexp.MustCompile("^```[^\\s]{0,20}\\s*\\n")
var closeFenceRE = regexp.MustCompile("\\n?```\\s*$")
```

Both patterns are **non-zero-width** under all inputs: `openFenceRE` requires at least the three backticks plus a newline (4 chars); `closeFenceRE` requires the three backticks at end-of-string. Neither can match a zero-length substring, so `ReplaceAllString` with an empty replacement is safe and terminates. This was validated by reasoning — there is no `*` or `?` quantifier on a position-only anchor and no alternation with a possibly-empty branch.

### 9.3 Title Derivation

```
title = firstNonEmptyLine(prompt)
title = strings.TrimSpace(title)
if len(title) > 50 { title = title[:50] }  // byte-safe because we later validate UTF-8 boundaries
if title == ""     { title = "Untitled melody" }
```

Byte slicing at 50 can split a multi-byte rune. The implementation walks back to the last rune boundary using `utf8.RuneStart` before committing the cut. Empty prompt → guard in handler returns 400 before this runs, so `"Untitled melody"` is only triggered when the prompt is exactly whitespace that bypasses the trim (not possible) — it is kept as defensive fallback.

### 9.4 Duplication Title Suffixing

```go
const copySuffix = " (copy)"
const maxTitle   = 200

newTitle := src.Title + copySuffix
if len(newTitle) > maxTitle {
    base := src.Title
    // truncate base on a rune boundary
    budget := maxTitle - len(copySuffix)
    if budget < 0 { budget = 0 }
    base = truncateRunes(base, budget)
    newTitle = base + copySuffix
}
```

**Edge cases:**

| Source title | Result |
|--------------|--------|
| `""` | `" (copy)"` — 7 chars, valid |
| `"Hello"` | `"Hello (copy)"` |
| `"Hello (copy)"` | `"Hello (copy) (copy)"` — suffix is re-applied, intentional |
| A 200-char title | Source truncated to 193 chars (rune-aligned) + `" (copy)"` = 200 chars |
| A 198-char title with a 2-byte rune at position 192 | Base truncated to 192 bytes (rune boundary before 193) + `" (copy)"` = 199 bytes |

### 9.5 Listing with Empty Collection

`ListMelodies` always returns `[]store.Melody{}` (non-nil, zero-length). The handler marshals this to `[]`, never `null`. Consumer JS treats both as "no melodies" but the contract is explicit.

## 10. Design Decisions and Rationale

1. **abcjs from CDN, not bundled** — abcjs-basic-min is ~300 KB. Loading from `cdn.jsdelivr.net` keeps the Docker image small and leverages browser caching across demo apps. Version is pinned (`@6.4.4`) so the UI is deterministic. Trade-off rejected: vendoring abcjs into `internal/templates/static/` would bloat the binary and complicate the embed. If `cdn.jsdelivr.net` is blocked in the user's network, the staff viewer fails to render but the ABC text editor and API continue to work.

2. **Embedded HTML template** — the entire SPA is one Go template file embedded via `//go:embed`. Matches the template pattern and keeps the binary self-contained.

3. **Single SPA, no framework** — plain HTML + vanilla JS (ES2020). Avoids a frontend build step. The UI is small enough to fit in ~500 lines of JS.

4. **Named Firestore database (`music-generator`), not `(default)`** — keeps projects cleanly isolated when multiple demo apps share a project, and matches the template's firestore-database-per-app convention.

5. **BulkWriter over Batch()** — `client.BulkWriter` is the current Google-recommended path for multi-doc writes. `Batch()` is deprecated, capped at 500 ops, and has no adaptive rate-limiting. Even though v1 only uses BulkWriter in `DeleteAllMelodies` (admin/test only), this is enforced codebase-wide for consistency.

6. **Root route `GET /{$}`, not `GET /`** — prevents the index handler from swallowing 405s on wrong-method API calls. See §6.

7. **Dependency injection via interfaces** — `store.Store` and `openai.Generator` are interfaces. Handlers accept them via the `Server` struct. Tests inject fakes; production injects `FirestoreStore` and `OpenAIGenerator`.

8. **Prompt sent to OpenAI** — a strict system prompt (below) instructs the model to return only valid ABC, no prose, no markdown. This maximises the happy-path hit rate; the fence-stripper in §9.1 is defence-in-depth.

9. **OpenAI system prompt (verbatim):**

   ```
   You are an expert music composer specialising in ABC notation.
   When the user describes a musical idea, produce a complete, valid ABC
   notation tune that realises that idea.

   Strict output rules:
   - Output ONLY the ABC notation.
   - No markdown, no code fences, no explanations, no commentary.
   - The tune MUST start with an X: reference number line and include
     T: (title), M: (meter), L: (default note length), and K: (key)
     header fields before any music.
   - Use multiple V: voices when the user asks for multiple voices.
   - Keep the tune under 32 bars unless the user requests more.
   - Use standard ABC 2.1 syntax that renders in abcjs.
   ```

   Stored as a Go `const systemPrompt = `...`` in `internal/openai/client.go`.

10. **OpenAI call parameters** — `Model: "gpt-4o"`, `Temperature: 0.8`, `MaxTokens: 2000`, `N: 1`. A context deadline of 60 seconds is applied by the handler to bound tail latency.

11. **No pagination in v1** — melody lists are expected to stay under ~100 entries per demo. If usage grows, add `?limit=&cursor=` params; Firestore supports cursor pagination natively via `StartAfter`.

12. **OpenAI key from Secret Manager** — the key is stored in Secret Manager and mounted as env var `OPENAI_API_KEY` in Cloud Run via the `secret` block of `google_cloud_run_v2_service`. The runtime SA has `roles/secretmanager.secretAccessor`. The secret is created **out-of-band** (manual step) to keep it out of Terraform state and git.

13. **Graceful degradation** — Firestore and OpenAI errors never crash the service. They produce structured JSON errors. Panics in handlers are recovered by middleware and returned as 500.

14. **Versioning** — `VERSION=1.0`, commit count appended by CI. Image tags `v1.0.42`, env var `APP_VERSION=v1.0.42`.

15. **Terraform 1.6+** — required by `rules/template-standards.md` for variable-in-import support.

16. **GCR (`gcr.io`) not Artifact Registry** — per the Cloud Run template standard in `rules/naming-conventions.md`.
